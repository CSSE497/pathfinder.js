requestCommodityTransit(cluster, start, destination, parameters, callback)

Requests transportation for a physical entity from one geographical location to another. This will immediately route a vehicle to pick up the commodity if one is available that can hold the commodities parameters within the vehicles capacity.
Parameters:
Name Type Description
cluster number The id of the cluster to request commodity transit within
start object The starting location of the commodity as {"longitude":x, "latitude":y}
destination object The destination for the commodity as {"longitude":x, "latitude":y}
parameters object The quantities of your application's routing calculations. The set of parameters needs to be defined and prioritized via the Pathfinder web interface in advance
callback function This function will be called exactly once with the created commodity object.
